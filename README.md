Cyclone Detection System (Manual Input)

A lightweight Python-based diagnostic tool designed to evaluate local weather conditions and provide early warnings for cyclonic activity based on barometric pressure and wind speed thresholds.📌 OverviewThis project serves as a Rule-Based Decision Support System. Instead of relying on a constant internet connection or external APIs, it allows users in remote or low-connectivity areas to manually input local sensor data to determine the severity of an approaching storm.🚀 FeaturesDual-Parameter Validation: Uses both Atmospheric Pressure ($hPa$) and Wind Speed ($km/h$) to reduce false positives.Condition Analysis: Incorporates qualitative weather states (Sunny, Windy, Rainy) to provide context-aware alerts.User Safety Protocols: Generates specific action items (e.g., "Evacuate" vs. "Monitor") based on the calculated risk level.Error Handling: Built-in protection against non-numerical inputs to ensure system stability during critical use.📊 How It Works (The Logic)The system triggers a Cyclone Warning based on the following meteorological criteria:ParameterThresholdSignificancePressure$< 995$ hPaIndicates a strong low-pressure system typical of cyclones.Wind Speed$> 60$ km/hMeets the minimum criteria for a "Cyclonic Storm" classification.WeatherWindy/RainyConfirms the physical manifestation of the pressure drop.🛠️ RequirementsPython 3.xNo external libraries (uses standard Python input and math logic).
